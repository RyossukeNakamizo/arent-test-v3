# 回答不能時サポートガイド

## 3段階サポート

### Level 1: エビデンス付き選択肢提示

技術選定・方式が未定の場合、プロジェクト文脈から候補を2〜3個提示する。

例（APS Viewer バージョン選定）:
- (a) Viewer v7 (SVF2) — 最新・パフォーマンス優位・公式推奨
- (b) Viewer v6 (SVF) — 実績豊富・情報多い・v7移行予定あり

### Level 2: 仮置き＋フラグ

選択肢から決められない場合、仮置きとして設計に組み込む。

- 仮置き記法: `{内容}（**【仮置き】要検証**）`
- 仮置き上限: CLAUDE.md内で最大3項目
- 仮置き不可: 禁止事項（Q6）・守るべきルール（Q5）

仮置き項目は `docs/templates/implementation-confirmation.md` の仮置き解消チェックリストで追跡する。

### Level 3: スコープ縮小の提案

仮置きすら困難な場合、ミニPOCを提案する。

例: 「APS Viewer v7 で dbId 取得が可能か、まず最小コードで検証しませんか？（所要: 30分）」
