# ヒアリングガイド — 指摘管理ツール

## 設計開始時の確認事項

設計書作成前に以下を一問一答で確認する。文脈から明らかな項目はスキップ可（推定内容を宣言）。

### 必須確認

1. **対象モデル**: APS URN・フロア数・要素数の確認
2. **指摘カテゴリ**: Quality / Safety / Construction / DesignChange 以外に追加が必要か
3. **位置タイプ**: 部材指摘(dbId) / 空間指摘(worldPosition) の比率・優先度
4. **写真要件**: 是正前/後の必須性・最大枚数・ファイルサイズ上限
5. **状態遷移**: Open → InProgress → Done で十分か。差し戻し（Done → Open）の要否
6. **ユーザー権限**: POCは単一ユーザー前提。将来のロール設計は 8.6 で検討済み

### 任意確認

7. **フィルタ要件**: 一覧のソート・フィルタ条件（ステータス別・カテゴリ別・日付範囲）
8. **通知要件**: 指摘登録時のSlack通知等（POCスコープ外）
9. **帳票出力**: 指摘一覧のCSV/PDF出力（POCスコープ外）

### 未定時の対応

「未定」回答時は `docs/references/support-guide.md` の3段階サポートに従う。
